What’s New in BridgePoint 4.1.9

Release Notes

 

xtUML eXecute

·         Referential attribute display in the Variables view is fixed to now show the proper value

User Interface

·         Constants may now be accessed via inter-project references.

·         When creating new classes, the key letters of the class are now automatically set to the class name with spaces removed.

·         There are major enhancements to the display of tooltips on xtUML diagrams.  While a tooltip is shown, a user can click inside the tooltip to give it focus and make it sticky.  The tooltip is displayed in a resizable and movable window that supports scrolling and text wrap.  In addition, multiple tooltips may be displayed at the same time.  Once a tooltip has received focus and become a managed window, it is displayed until it is manually closed by the user.

·         Command line build with the -prebuildOnly flag, and a -project specification no longer requires a DISPLAY when run on Linux.  This feature allows the build to run in a truly headless manner.  For example:

 $ CLI Build -project <name> -preBuildOnly

·         There have been significant improvements to model compare and merge:

o    Improved compare and merge functionality when using git

o    Multiple enhancements for state machine merging

o    Graphical elements no longer used during merge

xtUML Translate

·         Fixed the generation of polymorphic event name definitions so they cannot clash with local event names

·         Due to 16 bit counters, instance memory was not being completely cleared at delete time and at dynamic allocation time.  This is fixed.
