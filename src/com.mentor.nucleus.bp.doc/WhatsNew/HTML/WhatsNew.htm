What’s New in BridgePoint 4.0.7

Release Notes

 

xtUML Translate

The BridgePoint C model compilers contain major enhancements that significantly reduce translation time and memory usage.  The new translation process is integrated seamlessly with the existing functionality.  Users will see added informational messages (in red text) in the Console view during translation.  The performance and memory gains make the translation much more robust for large models.

xtUML eXecute

A number of fixes and enhancements were made to xtUML eXecute:

 

·         Fixed execution of synchronous operations when using Verifier Interface to Realized Components

·         Made the execution of multiple identical configurations more robust

·         Fixed comparison of attributes of type component_ref

·         Improved the consistency of lists of interface signals is various views within the tool

·         Improved the execution control provided by the “Suspend” and “Step” tools within an execution session

·         Improved realized component and interface library unloading on session termination

Document Generation

The output of document generator includes additional formatted information drawn from the model. 

Model Merge

The model merge capability inside the tool has been enhanced to improve the capability of State Machine merges.

External Entities

Two new features were added for External Entities.

 

First, a tool is provided to include the built-in External Entities Architecture (ARCH), Logging (LOG) and Time (TIM).  Simply select any package (typically a top-level package specifically for EEs) and choose “Add built-in External Entities” from the context menu on the package.

 

 

Second, the tool now includes support for realized External Entities in the same manner as existing support for realized Components:

 

 

Combined with inter-project references, users now have the ability to create EEs in a single project, provide a single Java implementation for the EE, and re-use that implementation across many projects.
