Undo and Redo

The xtUML Modeling perspective provides Undo and Redo stacks for each xtUML model. For example, the following illustration shows an xtUML project namedMicrowave with a single xtUML model MicrowaveOven.

In this illustration there is one Undo and one Redo stack attached to the MicrowaveOven model.  If a new xtUML model were to be created or imported into the Microwave project then new, separate undo and redo stacks would be created for that model also.

Undo and Redo stacks are available from Canvas Editors, Outline Views, and - when these editors or views are in focus - from Edit on the menu bar.  The default control key combination for Undo is defined as control-Z and the default control key combination for Redo is control-Y.

NOTE: Because Undo and Redo stacks are model-wide the results of an Undo or Redo may not be immediately visible.  For example, if an Instance State Machine Canvas Editor is currently in focus and the last change on the Undo stack represents the deletion of some other state machine, the results of executing the Undo will not be visible unless the Model Explorer or Outline views are expanded so that they display the affected state machine.

NOTE: There are two types of xtUML model elements that are not eligible for Undo and Redo.  Because the Undo and Redo stacks are model specific, when an entire xtUML model is deleted the stacks for that model are no longer available.  xtUML models may be restored from the Local History.  In place of Undo and Redo, Model elements such as State Action and all Descriptions, that are normally modified using text editors are eligible for the Eclipse text editor default Undo Typing and Revert mechanisms.

Accessing the Undo and Redo stacks from Canvas Editors

To access the Undo or Redo stacks from a Canvas Editor, bring the editor into focus and then right-click on any part of the canvas to display the context menu.

 

 

Accessing the Undo and Redo stacks from the Outline View

To access the Undo or Redo stacks from the Outline view, bring the view into focus and then right-click inside the view to display the context menu.

 

Accessing the Undo and Redo Stacks from the Menu Bar

When either a Canvas Editor or the Outline view is in focus, Undo and Redo are available in the Edit menu of the menu bar.

Restoring Deleted xtUML Models

When an entire xtUML model is deleted from a project the Undo and Redo stacks for the model are also deleted.  Models deleted in this manner may be restored from the Local History. 

To do this, change to the EDGE Projects perspective, navigate to and select the models directory of the project that contained the deleted model.  Right-click and select Restore from Local History… from the context menu.

The Restore from Local History dialog box will pop up.  Select the models you would like to restore and then press the Restore button.

The model is restored and again accessible from the xtUML Modeling perspective.

Using the Undo and Revert Features of Text Editors

In place of Undo and Redo, model elements such as State Action and all Descriptions that are normally modified using text editors are eligible for the Eclipse text editor default Undo Typing and Revert mechanisms.  These mechanisms are available from the context menu of each text editor.
